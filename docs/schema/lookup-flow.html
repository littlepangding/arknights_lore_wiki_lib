为什么是这个形状：agent 的工作是『不把整个语料塞进上下文』就能回答问题。整个流程要把范围 收缩两次（解析名字 → 借出场关系 / storyset 缩小）才开始阅读，并按代价从小到大读（摘要 < 分节 < 原文）。三种有色徽章标出了 v1 可能让你失望的地方：需注意 = 诚实承认的已知局限；缺口 = 『v1 里只能兜底或者根本没有』。紫色的锚点是你应当 fall back 到 的高置信源（不是要 fall back 离开 的）—— storysets.json 给确定性出场，card.json 给事实核验。
